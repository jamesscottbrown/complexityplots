About Why complexity plots?

The trouble with existing plots

One often wishes to measure the performance of algorithms or piece of "black box" software in terms of their algorithmic complexity. However, using traditional line graphs is problematic when attempting to visualise the complexity of such measurements. The figure to the left shows a linear (left) and a logarithmic (right) plot of a set of measurements with known complexity classes. The x-axis represents the input size, and the y-axis represents the measurement (e.g. runtime, energy usage, etc).

It is easy to observe from the above figure that judging the complexity classes is difficult on both linear and logarithmic scales. Not only is it difficult to judge the complexity classes in each case, but additionally the varying constant components of the measurements can render some measurements invisible (for example, the n² class is not visible in the linear plot).

A new kind of plot

It's clear that simply plotting measurements onto a line plot does not produce visualizations that are amenable to judging complexity. However, what if we can provide a new kind of spatial mapping to clearly present the varying complexity classes of algorithms with respect to their increasing input size? Well, it turns out that we can do just that. By using linear regression to obtain the characteristic properties of a set of data, a new kind of spatial mapping can be defined that maps input size verses measurement to input size verses complexity class. This is a complexity plot.

The complexity plot to the right shows, for varying input sizes on the y-axis, the complexity classes (on the x-axis) of a set of commonly-used sorting algorithms. It is now easy to observe the complexity classes of our measurements as they gradually 'settle down' and converge with larger amounts of data. In this case, the data size is defined as the size of the input array to sort.

More information

The complexity plot was created for a paper accepted to Eurovis 2013. Its authors are based at Oxford's University's e-Research Centre: Jeyarajan Thiyagalingam, Simon Walton, Brian Duffy, Anne Trefethen, and Min Chen.

For more details on the complexity plot visualization, including detail on the spatial mapping and complexity estimation techniques, please read our paper: Complexity Plots (PDF). You can reference our work using this bibtex entry.
