Data Formats For use with complexity plot creation

Preparing your data

For each measurement, you (or your code) should generate CSV data containing two columns: input size and the measurement recorded for the input size. As shown to the left, we allow for the first line to be a nonnumerical column header, which will be ignored during parsing.

Parsing issues

Should our server encounter issues in parsing, you will notice a warning next to the dataset in the data series list. Whilst our parser will make reasonable attempts to clean invalid values (such as empty strings or NaN's), please do ensure that the data is clean before uploading, containing only numerical values from row two onwards.

Example data files

For reference, we have provided three example data files used to generate the figure to the right, and they can be downloaded in a zip file here.

The files represent timing measurements from three sorting algorithms, shown below. You can download each file individually by clicking on the filename.

  • Bubble sort - bubble-sort.csv
  • Heap sort - heap-sort.csv
  • Insertion sort - insertion-sort.csv

Download the files, visit the creation app, and provide the files one-by-one to the creation app to see the complexity of each data series appear on the complexity plot!
